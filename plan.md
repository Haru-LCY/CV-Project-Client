# 当前改造计划

## 总结

项目已移除本地 MurasamePet Server 调用。客户端现在直接调用外部 API 生成人设、图片和桌宠回复，并把完整角色卡保存到 `config.json`。

## 已完成方向

- 删除本地服务端地址配置、角色接口调用和对话接口调用。
- 角色生成工作台只调用外部 API，不再做服务端兜底。
- 对话回复直接使用当前角色卡、用户称呼和本地对话历史调用 chat API。
- 角色名、人设、问候、图片 base64、情感图和生成选项保存到本地配置。
- 音频下载和播放逻辑移除。
- 桌面上下文事件不再采集或上传截图，只作为普通事件提示。

## 验证重点

- 没有本地服务端时，`uv run python pet.py` 能启动。
- 首次启动可打开角色工作台并生成预览。
- 应用角色后，`config.json` 保存完整角色卡，重启后可恢复。
- 输入文字或摸头后，回复语气与当前角色卡一致。
- 代码中不再出现本地服务端接口调用。
